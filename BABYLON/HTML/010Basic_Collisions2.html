简介

在动态场景中，物体移动并且和其他物体相交。为了取得最好的渲染效果，你需要知道你的网格什么时候和另一个网格相遇。在这个教程中，我们将探索碰撞系统是怎样工作的。

最终效果

怎么做 ?

这个教程将向你展示两种碰撞检测的方法：第一种是在两个网格相遇时触发一个碰撞事件，另一种是在一个网格和一个单点之间进行碰撞检测。

我们将讨论上面的场景。前两个球将会与旋转的平面相碰，最后一个球将会与一个单点相撞。在你建立这个基本场景之后，再继续阅读来学习怎样检测碰撞。

  • 相交网格

这里的点是用来检测球和平面之间的相遇的。我们将使用“intersectsMesh()”函数，带有两个参数：需要检测的网格，和相交的精度。

if (balloon1.intersectsMesh(plan1, false)) {
    balloon1.material.emissiveColor = new BABYLON.Color4(1, 0, 0, 1);
    } else {
    balloon1.material.emissiveColor = new BABYLON.Color4(1, 1, 1, 1);
    }

为了避免检测一个网格的所有细节的高昂计算成本，Babylon引擎围绕着物体建立了一个边界盒，并且在这个边界盒与相撞的物体之间检测相交。这里是一个边界盒的例子：

但是边界盒可以具有更多或更少精确度，这就是我们使用第二个参数的原因。简单来说，如果这个参数设为true（默认为false），那么这个边界盒是紧贴着网格的（OBB包围模式），但是计算成本更高。请记住这种模式的边界盒在网格按照某种角度旋转时是非常有用的。

所以请在选择之前考虑你所需要的碰撞细节.

如果你想要了解更多有关第二个参数的信息，你可以访问维基百科网页查找，尤其是关于AABB和OBB模式http://en.wikipedia.org/wiki/Bounding_volume

  • 点相交

另外一个你可以使用的函数是 “intersectsPoint()”通过一个特殊点,像这样:

var pointToIntersect = new BABYLON.Vector3(10, -5, 0);
    if (balloon3.intersectsPoint(pointToIntersect)){
    balloon3.material.emissiveColor = new BABYLON.Color4(1, 0, 0, 1);
    }

我在场景中定义了一个精确的点，如果我们的球与这个点相交，无论是球上的哪个位置，那么这个事件将会被触发，然后我们改变这个球的颜色。

你可以尝试这个教程中使用的代码。。。通过访问我们playground中的一个示例.
